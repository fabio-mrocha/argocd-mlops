# argocd-mlops

GitOps repository for bootstrapping an MLOps platform on Kubernetes using [Argo CD](https://argo-cd.readthedocs.io/). A single "app of apps" `Application` points at this repo and drives Argo CD to deploy and manage every add-on declared here — no manual `kubectl apply` needed after the initial bootstrap.

## Stack

Deployed and managed as Argo CD `ApplicationSet` generators:

| Component | Type | Namespace | Version |
|---|---|---|---|
| [cert-manager](https://cert-manager.io/) | Helm chart | `cert-manager` | v1.20.3 |
| [Argo CD](https://argo-cd.readthedocs.io/) | Helm chart | `argocd` | 10.1.2 |
| [Apache Airflow](https://airflow.apache.org/) | Helm chart | `airflow` | 1.22.0 |
| [MLflow](https://mlflow.org/) | Helm chart | `mlflow` | 1.11.2 |
| [Kubeflow Pipelines](https://www.kubeflow.org/docs/components/pipelines/) (platform) | Kustomize | `kubeflow` | 2.16.1 |
| [Kubeflow Pipelines](https://www.kubeflow.org/docs/components/pipelines/) (cluster-scoped resources) | Kustomize | `kubeflow` | 2.16.1 |

## Repository layout

```
gitops/
├── apps-of-apps/
│   └── applications.yaml        # Root Application; applied once by hand to bootstrap everything else
├── applications/
│   ├── addons.yaml               # ApplicationSet generating one Application per values-*.yaml (Helm charts)
│   └── addons-kustomize.yaml     # ApplicationSet generating one Application per manifests-*.yaml (Kustomize)
└── addons/
    ├── charts/                   # Chart coordinates (name, namespace, repo url, version) — one file per Helm add-on
    ├── helms/                    # Helm values files, one per add-on, referenced by charts/values-*.yaml
    └── kustomize/                # Kustomize source coordinates (repo, revision, path) — one file per Kustomize add-on
```

To add a new Helm-based add-on: drop a `values-<name>.yaml` in `gitops/addons/charts/` (chart coordinates) and a matching Helm values file in `gitops/addons/helms/`. The `addons` `ApplicationSet` picks it up automatically on the next sync.

To add a new Kustomize-based add-on: drop a `manifests-<name>.yaml` in `gitops/addons/kustomize/` describing the source repo/path. The `addons-kustomize` `ApplicationSet` picks it up automatically.

## Prerequisites

- A Linux host (or VM) for the k3s node
- `kubectl` and `helm` installed locally
- Network access to the container/chart registries used above

## Bootstrap

### 1. Install k3s

```bash
curl -sfL https://get.k3s.io | K3S_KUBECONFIG_MODE="644" INSTALL_K3S_EXEC="--cluster-cidr=10.42.0.0/16 --service-cidr=10.43.0.0/16" sh -
```

### 2. Clone this repository

```bash
git clone https://github.com/fabio-mrocha/argocd-mlops.git
```

### 3. Install Argo CD

```bash
kubectl create namespace argocd

helm repo add argo https://argoproj.github.io/argo-helm

helm template argo-cd argo/argo-cd --version 10.1.2 --namespace argocd \
  -f argocd-mlops/gitops/addons/helms/values-argo-cd.yaml | kubectl -n argocd create -f -
```

### 4. Retrieve the Argo CD admin password

```bash
kubectl -n argocd get secret argocd-initial-admin-secret -o jsonpath="{.data.password}" | base64 -d
```

### 5. Apply the app-of-apps

This is the only manifest you apply by hand — it bootstraps the `addons` and `addons-kustomize` `ApplicationSet`s, which in turn deploy and reconcile everything listed in the [Stack](#stack) section.

```bash
kubectl -n argocd create -f argocd-mlops/gitops/apps-of-apps/applications.yaml
```

### 6. Access the Argo CD UI

```bash
kubectl port-forward -n argocd --address='0.0.0.0' service/argocd-server 8080:443
```

Then open [https://localhost:8080](https://localhost:8080):

- **user:** `admin`
- **pass:** `<password from step 4>`

## Sync policy

All Applications generated by the `ApplicationSet`s use automated sync with:

- `prune: true` / `selfHeal: true` — the cluster is continuously reconciled to match this repo
- `PruneLast=true` — new resources are created before old ones are pruned
- `ApplyOutOfSyncOnly=true` — only out-of-sync resources are touched on each sync
- `CreateNamespace=true` — target namespaces are created automatically
